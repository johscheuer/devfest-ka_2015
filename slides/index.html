Containers

- Simple isolation - Functionallity similiar to VM's - Light weight - Fast - Package dependencies - Simple deployment
- Most known Container tool - Has own Deployment tools like machine, swarm, compose - Swarm production ready can be used to roll out other cluster scheduler - Not as smart as Kubernetes or Mesos

Simple Dev setup

- Dev's like Docker because fast Dev env - Run fast test - Spin up in some seconds - Better/Faster than VM's like vagrant - Widely used in Dev
- Greek for pilot or helmsman of a ship - Open Source cluster manager from Google - Managing containerized applications across a cluster of nodes - Easy Service deployments, updates and (auto) scalability - Compute resource limits - Object count limits (later) - namespaces (logical speration)

Pods

- Group of containters - Share Resources e.q. CPU/Storage/Network - Can communicate over localhost - Defined in an template file - Kubernetes Scheduler sched. on nodes - Reusable Unit - Repeatable deployment

Replication Controller

- Logical unit - Keeps the desired number of pods running - Can scale Pods - Can restart pods - Fine graned scaling - Direct controll to number of running Pods - Lifecycle management of containers

Service

- Stable address - Decoupled from Controller - Independent from Client implementation - Allows dynamic port mapping on the client/pod - Enable to find and consume services
